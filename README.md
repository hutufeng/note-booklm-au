# 书籍知识点视频教程自动生成工具

> 基于 [notebooklm-py](https://github.com/teng-lin/notebooklm-py) 非官方 API，将**任意书籍**自动生成按章节组织的**视频讲解**和**交互式课后测试题**，最终打包为**可单机离线使用的 HTML 网页教程**。
>
> 支持**理工科**（数学/物理/化学/计算机）和**文科**（历史/哲学/文学/社科）两种模式，使用不同的 Prompt 模板。

---

## 功能概述

| 功能 | 说明 |
|------|------|
| 📁 项目管理    | 列出/新建 NotebookLM 项目，管理上传资源（幂等，重复运行安全）|
| 📋 AI 目录生成 | 让 NotebookLM 从书中自动生成三级知识点目录，存入**NotebookLM 笔记**|
| 🔁 断点与同步  | 目录及生成进度优先使用云端已生成的资源，精准拉回本地（跨终端同轨） |
| 🎯 手动选择    | 按章/节/知识点灵活勾选要生成的范围，支持并发拉取组合选择 |
| 🎬 媒体资源    | 全息生成视频、音频播客（匹配 PPT 讲解）、演示文稿（幻灯片）与信息图|
| 📝 学习材料    | 高质量生成深度学习报告、数据表格、互动巩固 / 拔高测试题及多视角闪卡 |
| 🛡️ 智能防呆机制 | **演示文稿优先级极高**：生成音频时，强制优先对齐已有的演示文稿内容结构进行解说 |
| ⚡ 限额与安全  | 免费/Pro 定制化并发与防限流控制机制（自动防风控） |
| 💾 健壮的续传  | 中断后下次自动读取 `progress.json` 及云端实际存储情况进行无损续传 |
| 🧹 智能任务管理 | 独立「任务管理」面板，提供清理本地僵尸文件、绕过 SDK 误判的强制拉取等进阶能力 |
| 🌐 沉浸式网页  | 自动构建单机离线的响应式网页教程（优化了 PDF 调用体验与多折叠面板默认展示方式）|

## 快速开始

### 1. 安装 conda 环境

```powershell
conda env create -f environment.yml
conda activate notebooklm
```

### 2. 配置 Playwright 内容（请勿使用 Edge）

强烈建议使用 Playwright 独立下载的 Chromium 浏览器，**不要复用系统自带的 Edge**，因为 Edge 会导致不可预料的强缓存报错（如网页数据过期）及 Cookie 文件锁冲突。

```powershell
# 下载独立的 Chromium（与系统浏览器完全隔离，约 140MB）
playwright install chromium
```

### 3. 登录 NotebookLM

```powershell
notebooklm login
```

> 浏览器打开后，用 Google 账号登录，登录成功后返回终端按 **Enter** 保存会话。

### 4. 运行主程序

```powershell
conda activate notebooklm
cd c:\Users\hutu_\notebooklm
python book_generator.py
```

## 使用流程

### 主菜单

```
[1] 配置项目   → 选择/新建 NotebookLM 项目，上传书籍文件，设置书名和类型
[2] 管理目录   → 查看/生成/重新生成三级知识点目录（存入 NotebookLM 笔记）
[3] 开始生成   → 选择知识点范围，设置账号类型，生成视频和测试题
[4] 生成网页   → 将已下载的素材打包为可单机使用的 HTML 网页
[5] 任务管理   → 二次补救下载、清理本地残留文件或清除云端记录释放配额
[0] 退出
```

### Step 1 — 配置项目（选项 1）

- 选择已有的 NotebookLM 项目或新建
- 查看已上传资源，按需上传书籍 PDF/文本文件
- 输入书名（如：《什么是数学》、《人类简史》）
- 选择书籍类型：
  - `[1] 理工科` — 强调公式推导、定理证明、计算步骤
  - `[2] 文科` — 强调论点梳理、历史背景、批判性思考

### Step 2 — 管理目录（选项 2）

```
[1] 查看/加载   → 自动在 NotebookLM 笔记中查找已有目录；没有则询问是否生成
[2] 展示目录    → 展示三级目录并标注各知识点完成状态
[3] 重新生成    → 覆盖已有目录（可自定义指定 1~3 级目录深度）
[4] 删除目录    → 一键销毁本地缓存 和/或 NotebookLM 目录笔记
```

**目录存储机制：**

```
生成后同时写入 ──→ [NotebookLM 笔记] 标题：「[课程目录] 书名」
                └→ [本地缓存] curriculum/curriculum_书名.json
加载时优先级 ──→ NotebookLM 笔记 > 本地缓存
```

### Step 3 — 生成内容（选项 3）

选择知识点范围：

| 输入 | 含义 |
|------|------|
| `all` | 全部知识点 |
| `ch01` | 整个第一章 |
| `ch01_s02` | 第一章第二节 |
| `ch01_s02_t01` | 单个知识点 |
| `ch01,ch08_s03` | 组合（逗号分隔）|

选择账号类型：

| | 免费账号 | Pro 账号 |
|--|---------|---------|
| 并发任务 | 1个（串行）| 3个 |
| 测试题数量 | 10题 | 13题（含拔高）|
| 冷却时间 | 30秒 | 10秒 |

### Step 4 — 生成网页（选项 4）

生成 `output/website.zip`，解压后用浏览器打开 `website/index.html`，无需网络。

### Step 5 — 任务管理（选项 5）

这是一个非常强大的后悔药与控制面板，分为两个大区：
- **查看与删除**：
  - 可以查看只包含真实任务的精简记录榜单。
  - **[4] 删除云端 Artifact**：当 NotebookLM 配额快用完时，删除历史任务记录。
  - **[5] 删除本地文件**：当需要重刷或者清理空间时，强制粉碎本地各类多媒体文件。
- **二次下载（[7]）**：
  - 智能检索丢失的文件，分为【仅补充未下载】和【全部覆盖强制下载】双模式。
  - 内置了对抗 SDK BUG 的**强制无视状态**暴力拉取选项（专门解决部分文件 API 状态不更新的问题）。

## 书籍类型 Prompt 差异

可通过临时加载的 `custom_prompts.json` 针对不同的物料（文稿、播客、信息图、报告等）进行全自定义配置。默认模板特点：

### 理工科模板（STEM）
- **讲解媒体**：概念引入 → 规范定义 → 推导/证明（不跳步，注重画外音推导）→ 数值例题
- **视觉图表**：以梳理框架流程、数据对比、逻辑拆解为目标的数据表格与信息图
- **自测材料**：以填空、辨析、计算和核心定理为主

### 文科模板（Humanities）
- **讲解媒体**：时代背景 → 核心论点 → 思想碰撞与多视角交锋 → 留白思考
- **视觉图表**：展示核心思想长河演化、事件相关性时间轮廓信息图
- **自测材料**：概念理解、观点判断、论点评析与流派对比

## 注意事项

- **API 风险**：`notebooklm-py` 使用非官方 API，Google 随时可能修改接口
- **Source 索引**：上传文件后需等待 NotebookLM 完成索引才能生成目录
- **会话过期**：遇到 "Not logged in" 错误时重新运行 `notebooklm login`
- **视频语言**：已设置 `language=zh_Hans`，视频和测试题均为简体中文
- **目录书名区分**：不同书籍会生成不同的缓存文件，切换书籍请在选项 1 中更改书名

## 账号安全说明

### 封号风险：低（正常使用不会被封）

本工具使用**真实浏览器 Cookie 会话**登录，行为与正常用户操作完全一致，不是爬虫。
Google 主要通过**请求频率和行为模式**识别自动化，已内置冷却策略规避。

| 风险行为 ❌ | 安全行为 ✅ |
|-----------|----------|
| 并发提交大量任务 | 串行执行 + 冷却间隔（已内置）|
| 24 小时持续不间断运行 | 工作时段使用，用完停止 |
| 将 `storage_state.json` 分享给他人 | 仅个人使用自己的账号 |
| 手动改小冷却时间为 0 | 保持默认冷却时间不变 |

### 遇到限流如何处理

程序输出 `RPCError` 或任务持续超时时，**立即停止**，等待 2~4 小时后继续：

```powershell
# 断点续传，自动跳过已完成任务，无需重新登录
python book_generator.py

# 如提示未登录，重新认证
notebooklm login
```

### 建议使用专用学习账号

> 因使用非官方 API，建议**不要用主力工作账号**，开一个专用学习 Google 账号更为稳妥。
